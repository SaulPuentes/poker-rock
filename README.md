<p>The application will wait for 4 players to be logged in order to start the game. If one payer is missing, the game will not start.</p>
<h2><a id="user-content-setup-table" class="anchor" aria-hidden="true" href="#setup-table"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Setup table</h2>
<p>When the game starts, the table is initialized. There are 4 players, with their respective session credentials, and the table, which is a variation of the object of the player.</p>
<h2><a id="user-content-rounds" class="anchor" aria-hidden="true" href="#rounds"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Rounds</h2>
<ul>
<li>There are up to 3 rounds. There will be a counter for each round, will it be modified after the players movements.</li>
<li>During each Round, the numbers of cards held by the table will change. During the first round, the table will only have 3 cards, in the second one it will have 4 and in the third, and last, one it will have 5 cards.</li>
</ul>
<h2><a id="user-content-add-card" class="anchor" aria-hidden="true" href="#add-card"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Add card</h2>
<ul>
<li>In an array, the indexes of all the 52 cards will be stored. When the table is initialized, the 52 cards are âshuffledâ with a random function and stored in a new array.</li>
<li>The first 3 cards of the new array will pop in order to assign these cards to the table. When a new Round is about to begin, another card for that given array is pop again.</li>
<li>2 cards more, per player, will pop to be assigned to each user.</li>
<li>After this, the game is ready to begin.</li>
</ul>
<h2><a id="user-content-players-move" class="anchor" aria-hidden="true" href="#players-move"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Players Move</h2>
<ul>
<li>Each player has 4 possible moves:
<ul>
<li>Raise: The player can increase the bet. These points will be taken from the score of each user.</li>
<li>Fold. The player can standby and decide to not to play any further in the round.</li>
<li>Call. The player can equal the initial bet.</li>
<li>Quit. The player will logout from the application.</li>
</ul>
</li>
<li>The first player will make the initial bet.</li>
<li>The order of each turn will be assigned to the player at the right of the previous player.</li>
<li>Each player has a counter of 30 seconds to make their move, if the player does not make any move, the default move will be a Fold.</li>
<li>During each playerâs turn, the player can only make one of the 4 possible moves.</li>
<li>If a player makes a Raise, the other players will have to make a raise or call. There can not be a player whose bet is lower than the rest of the players.</li>
<li>When a player makes a Call or a Raise, their score will be decreased by the given amount, this will alter the collection from the database. So, if a player does not have enough points they can not make a Raise or a Call, only a Fold.</li>
</ul>
<h2><a id="user-content-end-of-round" class="anchor" aria-hidden="true" href="#end-of-round"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>End of round</h2>
<ul>
<li>In order to End a Round, all the players must have moved, in their last turn, as Fold. When all the players make a Fold, the Round counter will increase in 1. So the next Round can start.</li>
<li>The Table will call for another card and the players later can make a move again  repeating the mechanic.</li>
</ul>
<h2><a id="user-content-end-of-match" class="anchor" aria-hidden="true" href="#end-of-match"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>End of match</h2>
<p>When the Round is the last tone, the 3rd Round, and all the players make a Fold, the match will end, retrieving the data from all the players. Their current cards and score.</p>
